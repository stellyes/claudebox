## The Link

Both mesh networks (in computing) and collective memory (in communities) share the same fundamental architecture and the same failure mode. In a mesh network, each node relays what it receives to neighbors; the network is resilient above a percolation threshold and fragile below it. Collective memory works identically: knowledge relays through social transmission, and below a critical density of active transmitters, the knowledge stops propagating.

The connection is not metaphorical. The percolation threshold that governs whether a message propagates through a mesh network is the same mathematical structure that governs whether foraging knowledge, craft skills, or oral traditions survive in a community. Both are instances of the same underlying network dynamics.

## Evidence

**From computer networking**: In mesh networks, routing emerges without central coordination. Each device knows only its immediate neighbors. The network discovers paths dynamically. Resilience comes from redundancy: when individual nodes fail, traffic reroutes. Below the percolation threshold (too few active nodes), the network fractures into disconnected islands.

**From urban foraging**: Falling Fruit (2013) maps 1.9 million food locations contributed by people who never coordinated. Each contributor added local knowledge; the result propagates to all future users. When knowledge transmission stopped (industrialization of food supply), the underlying food resources persisted while the knowledge network fractured — exactly the below-threshold behavior of a sparse mesh.

**From oral tradition**: Aboriginal songlines represent 7,500-12,600 years of knowledge transmission — one of the longest continuous records in human history. They survived not through writing but through active relay: every generation learned and re-transmitted. The mechanism is identical to mesh networking: each "node" learns from neighbors and relays forward.

**From ecology**: Mycorrhizal networks (fungal connections between plant root systems) exhibit the same topology: resource-sharing without central control, fragile when node density drops below a threshold. The forest's collective memory of soil chemistry and seasonal patterns is encoded in the network's topology, not any single node.

## Implications

1. **The percolation threshold is a design parameter**: Communities that want to preserve knowledge need to maintain node density above the propagation threshold. This is a quantifiable target, not just a vague "keep traditions alive."

2. **Dark matter knowledge**: In any real community network, some knowledge clusters are below the propagation threshold (known in only 2-3 people) while others are robustly distributed. The total "knowledge capital" of a community includes both — but only the robustly distributed knowledge will survive attrition.

3. **Reconstruction has limits**: A mesh network can be rebuilt if enough nodes still exist. Collective memory can be reconstructed (as Falling Fruit demonstrates) if any nodes persist. But below a minimum — when all carriers have died or moved — the knowledge must be re-derived from scratch, paying Landauer's cost again.

4. **Shared maps as network amplifiers**: Tools that make knowledge externally visible (maps, wikis, oral performance traditions) effectively increase the relay density by converting private knowledge to relayable form. The map doesn't just record — it increases network robustness.
